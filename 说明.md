# 糖糖桌宠 (Desk Pet)

> 一个桌面虚拟主播宠物 —— 像素风角色常驻桌面，能聊天、能看你窗口、能主动搭话。

---

## 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 桌面框架 | **Tauri v2** | Rust 驱动的轻量桌面应用框架 |
| 前端 | **Vue 3** + **TypeScript** + **Vite** | 响应式 UI + 类型安全 |
| AI 后端 | 统一 OpenAI 兼容接口 | DeepSeek / OpenAI / Ollama 等 |
| 配置 | **YAML** (js-yaml) | CONFIG.yaml + CONFIG-DEV.yaml |
| 样式 | 原生 CSS + 像素风资源 | `image-rendering: pixelated` |
| 包管理 | **pnpm** | 快速、节省磁盘 |
| Rust 包管理 | **Cargo** | Rust 的构建系统 |
| 目标平台 | **Windows + macOS** | 双端完全适配 ✅ |

### 快捷键召唤 (v0.8+)

```
全局快捷键 (可配置，录制时自动禁用原快捷键) → Rust get_cursor_position()
                          │
                          ▼
                    App.vue handleShortcutToggle()
                          │
               ┌──────────┴──────────┐
               │                     │
          收回 (窗口可见)      弹出 (窗口隐藏/最小化)
               │                     │
      更新家位置(每次)          opacity:0 → show() → unminimize()
      光标原点缩放 1→0        窗口放光标处 (仅 clamp 边缘)
      (250ms ease-in)         光标原点缩放 0→1
      hide() 先隐藏          (350ms elastic)
      再清样式(零白窗)         playPopupSound()
      恢复家位置              focusInput() 聚焦输入框
      playRetractSound()
               │                     │
               └──────────┬──────────┘
                          │
          Dock 点击 → onFocusChanged → 屏幕中央淡入

macOS 窗口层级: ActivationPolicy::Accessory(LSUIElement) + MainMenu(24) + canJoinAllSpaces+fullScreenAux+stationary+ignoresCycle (iTerm 风格)
	设置窗口: alwaysOnTop: true + setTimeout 300ms → invoke("enhance_settings_window")(Rust: level 100 + orderFrontRegardless + makeKeyAndOrderFront)
              → 在所有桌面/全屏 App 上方悬浮，Dock 隐藏
```

- **弹出**：Rust `compute_popup_position` 在 Rust 侧完成 Cocoa→web 坐标转换 + clamp，前端直接用返回的 `(win_x, win_y)` 调 `setPosition`
- **收回**：Rust `get_cursor_position` 统一返回 web 坐标系（macOS 已做 Y 轴翻转 `web_y = 主屏高度 - cocoa_y`），前端用 `cursor - 窗口位置` 直接算 transformOrigin
- **收回**：先 `hide()` 再清除样式，零白色残影
- **窗口拖动**：`onMoved` 追踪 `lastMovedPos`（和 `setPosition` 同一坐标系，避免 `outerPosition()` 的物理/逻辑像素不一致）→ 仅 `emit("deskpet-moved")` 通知设置面板 → mouseup 检测手动拖动结束 → 用 `lastMovedPos` 保存 `userConfig.fixedPosition`
- **窗口缩放**：`onResized` → `ignoreResizeCount` 计数忽略程序化 setSize 触发的 resize（每次 setSize 前设 ignoreResizeCount=3），仅用户手动拖边缩放时保存 `userConfig.popupSize`。预览/恢复默认通过 `emit("deskpet-preview-size")` 协调 → App.vue 设置 `ignoreResizeCount` 防止 DPI 取整反馈循环
- **弹窗大小 DPI 防护**：`setWindowSize()` 记录目标值 + `ignoreResizeUntil` 时间基准（1秒保护）→ `onResized` 中检查时间戳，程序化 resize 不保存不发送实际尺寸（发送目标值避免反馈）。`setWindowPos()` 显式设 `lastMovedPos`，不依赖异步 `onMoved` 事件时序 → 确保收回/弹出位置一致
- **无白窗闪烁**：弹出前设 `opacity:0` 再 `show()`；收回后先 `hide()` 再清样式；`html,body{background:transparent}` 防浏览器默认白底
- **输入框聚焦**：弹出后自动 `focusInput()`，可直接打字
- **右键菜单**：选中文字后右键 → 自定义菜单（仅复制），无选中时禁用
- **Dock 隐藏**：`ActivationPolicy::Accessory`，桌宠不在 Dock 显示图标，纯后台浮窗
- **全局禁选/禁拖**：`user-select: none` + `img { -webkit-user-drag: none }` + `draggable="false"`，仅聊天消息区可选中文字
- **设置窗口**：`alwaysOnTop: true` + `setTimeout 300ms → invoke("enhance_settings_window")` → Rust: macOS level 100 + `orderFrontRegardless` + `makeKeyAndOrderFront` → 保存后不自动关闭，显示"已保存"提示 3 秒消失

## 配置系统

**所有配置集中在根目录 YAML，不在代码里写死常量。**

```
CONFIG.yaml                     ← 默认配置 (跟 git)
    │
    ├── CONFIG-DEV.yaml 存在 && enabled: true  → 完全替换，CONFIG 不加载
    ├── CONFIG-DEV.yaml 存在 && enabled: false → 忽略 DEV，走 CONFIG
    └── CONFIG-DEV.yaml 不存在                 → 走 CONFIG
         │
         ▼ Vite YAML Plugin (编译时 js-yaml 转换)
         │
    src/services/config.ts      ← 类型化 getter（9 组，全部 overrideOr() 动态读取 localStorage 覆盖值）
         │
         ├── aiConfig            → AI 端点/Key/模型/上下文数/fallback/默认人格
         ├── windowMonitorConfig → 停留秒数/防抖/冷却/同页冷却/额外延迟
         ├── aiLockConfig        → 生成锁安全超时
         ├── memoryConfig        → 最大记忆条数
         ├── notificationConfig  → 通知自动关闭延迟
         ├── desktopConfig       → Rust 轮询间隔/暂停额外时间（启动时 invoke 传递）
         ├── shortcutConfig      → 全局快捷键键值/修饰键 (macOS/Windows 分别配置)
         ├── loggingConfig       → 日志级别 (debug/info/warn/error)
         └── userConfig          → 弹窗模式/位置/大小/快捷键/音效分配 (localStorage 持久化覆盖 CONFIG.user 默认值)
```

**本地开发**：复制或创建 `CONFIG-DEV.yaml`，设置 `enabled: true`，改里面的参数调试。然后在 `.gitignore` 里忽略，不会提交。

## 日志系统

所有日志（前端 + Rust）**统一输出到终端**，也保留在浏览器 DevTools Console。

```ts
import { createLogger } from "@/services/logger";
const log = createLogger("模块前缀");   // 如 "AI"、"WinMon"、"Chat"

log.debug("调试细节");    // 仅 logging.level = debug
log.info("关键信息");     // debug|info
log.warn("警告");         // debug|info|warn
log.error("发生错误", e); // 永远输出
```

**输出格式**：`[HH:MM:SS.mmm] LEVEL [前缀] 消息`

**Rust 端**用 `rust_debug!` / `rust_info!` / `rust_warn!` 宏，格式一致，由 `log_message` command 接收前端日志并 `println!`。

**日志级别**在 `CONFIG.yaml` → `logging.level` 控制。DEV 配置默认 `debug`（全部输出），CONFIG 默认 `info`（生产模式显示 info+）。

## 启动说明

### 前置要求

- **Node.js** ≥ 18
- **pnpm** — `npm install -g pnpm`
- **Rust** — `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh`
- **macOS**: Xcode Command Line Tools — `xcode-select --install`

### 安装 & 启动

```bash
cd Desk-Pet
pnpm install                    # 安装前端依赖
pnpm tauri dev                  # 开发模式启动
pnpm tauri build                # 生产构建
```

### 配置

```bash
# 复制 DEV 配置模板（如不存在）
cp CONFIG-DEV.yaml.example CONFIG-DEV.yaml

# 编辑 DEV 配置，确保 enabled: true，填入 API Key
# 改其他参数（冷却时间、停留秒数等）也在这个文件
```

```yaml
# CONFIG-DEV.yaml
enabled: true

ai:
  apiKey: "sk-xxx"             # 本地 API Key
  # endpoint: http://localhost:11434/v1  # 本地 Ollama

windowMonitor:
  staySeconds: 20              # 调试：缩短等待
  cooldownSeconds: 3000        # 调试：缩短冷却
```

`enabled: false` 或删掉文件 → 回退到 `CONFIG.yaml`。

### macOS 权限

窗口标题监控需要**辅助功能权限**：
- 系统设置 → 隐私与安全性 → 辅助功能 → 允许"糖糖桌宠"
- 未授权时 AI 聊天/表情/模拟器仍可用，窗口监控不工作
